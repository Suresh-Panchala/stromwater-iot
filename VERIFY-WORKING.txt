================================================================================
TROUBLESHOOTING
================================================================================

Problem: Still seeing "No device data available"
Solutions:
  1. Wait 10 seconds for first data to arrive
  2. Press F5 to refresh browser
  3. Check backend terminal shows "Data stored successfully"
  4. Check simulator is still running

Problem: Dashboard shows but data doesn't update
Solutions:
  1. Check "Live" indicator - should be green
  2. Check browser console (F12) for errors
  3. Check WebSocket connection:
     - Backend should show "WebSocket server initialized"
     - Frontend should connect automatically

Problem: Charts are empty
Solutions:
  1. This is normal for first few minutes
  2. Charts need 24 hours of data to show trends
  3. Wait 5-10 minutes, charts will start showing data
  4. Or check "Device Detail" page for current data

Problem: Pump status not changing
Solutions:
  1. This is normal - pumps change based on water level
  2. Simulator varies water level automatically
  3. Watch for 30-60 seconds, status will change

================================================================================
WHAT'S HAPPENING BEHIND THE SCENES
================================================================================

Every 5 seconds:

1. Simulator generates random device data
2. Publishes to MQTT topic: devices/StromWater_Device_1/data
3. Mosquitto broker receives message
4. Backend MQTT client receives message
5. Backend stores data in PostgreSQL database
6. Backend broadcasts update via WebSocket
7. Frontend receives WebSocket message
8. Dashboard updates in real-time
